Here is the step-by-step blueprint to set up a completely free, automated Discord telemetry pipeline for FocusFlow from scratch. This requires zero accounts or manual actions from your users.

---

## Step 1: Create Your Private Discord Server & Webhook

You need a secure place where these crash logs will land.

1. **Open Discord** and click the **`+`** icon on the left sidebar to **Create a Server**. Name it something like `FocusFlow Dev`.
2. Inside your new server, hover over text channels and click the **`+`** to create a private channel named **`#prod-crashes`**.
3. Click the **Gear Icon (⚙️)** next to your `#prod-crashes` channel name to open **Channel Settings**.
4. Go to **Integrations** on the left menu, then click **Webhooks** (or **Create Webhook**).
5. Name the bot `FocusFlow Crash Bot` and click **Copy Webhook URL**.
* *Keep this URL safe and private—anyone with this link can send messages to your server.*



---

## Step 2: Set Up the Anonymous JSON Payload

Discord webhooks require data to be sent via an HTTP `POST` request formatted in a specific JSON structure. To keep FocusFlow completely privacy-focused, we will only collect non-PII technical details:

* The exception name and message.
* The system's Windows architecture/version.
* The clean, truncated stack trace.

---

## Step 3: Implement the Code in `CrashReporter.kt`

Open `CrashReporter.kt`. Locate your unhandled exception listener block (where the app currently hooks into thread failures to write logs to the local disk).

Add this `sendToDiscord` helper function and invoke it on a background thread inside your main crash handler. This ensures that if the user's internet is down or slow, the network call won't freeze their desktop or hold up the application exit.

```kotlin
import java.io.OutputStream
import java.net.HttpURLConnection
import java.net.URL
import java.nio.charset.StandardCharsets

object CrashReporter {
    // Paste your copied Discord webhook URL here
    private const val DISCORD_WEBHOOK_URL = "YOUR_COPIED_DISCORD_WEBHOOK_URL_HERE"

    fun sendToDiscord(errorMessage: String?, stackTrace: String) {
        // Run on a separate background thread so the UI/Kiosk never freezes during an exit
        Thread {
            try {
                val url = URL(DISCORD_WEBHOOK_URL)
                val conn = url.openConnection() as HttpURLConnection
                conn.requestMethod = "POST"
                conn.setRequestProperty("Content-Type", "application/json; utf-8")
                conn.doOutput = true

                // Discord has a 2000 character limit for regular text fields. 
                // We truncate the stack trace to 1500 characters to safely fit into the code-block embed layout.
                val cleanTrace = if (stackTrace.length > 1500) {
                    stackTrace.substring(0, 1500) + "\n... [Truncated for Length]"
                } else {
                    stackTrace
                }

                val safeMessage = errorMessage?.replace("\"", "\\\"") ?: "Unknown Exception"
                val safeTrace = cleanTrace.replace("\"", "\\\"").replace("\n", "\\n").replace("\r", "")

                // Construct an elegant Discord Rich Embed payload
                val jsonPayload = """
                {
                  "username": "FocusFlow Telemetry",
                  "embeds": [{
                    "title": "⚠️ Production Crash Detected (v1.0.5)",
                    "color": 16711680,
                    "fields": [
                      { "name": "Error Message", "value": "$safeMessage", "inline": false },
                      { "name": "OS Environment", "value": "${System.getProperty("os.name")} (${System.getProperty("os.version")})", "inline": true },
                      { "name": "Java Runtime", "value": "${System.getProperty("java.version")}", "inline": true }
                    ],
                    "description": "**Stack Trace:**\n```kotlin\n$safeTrace\n```"
                  }]
                }
                """.trimIndent()

                // Write bytes out to the wire
                conn.outputStream.use { os ->
                    val input = jsonPayload.toByteArray(StandardCharsets.UTF_8)
                    os.write(input, 0, input.size)
                }

                // Force connection to resolve and read response code (e.g., 204 No Content)
                val responseCode = conn.responseCode
                
            } catch (e: Exception) {
                // IMPORTANT: Keep this block completely silent. 
                // Catching network drops or timeouts here ensures the crash reporter never enters an infinite loop.
            }
        }.start()
    }
}

```

## Step 4: Wire It To Your Crash Handler Hook

Inside your primary uncaught exception hook (likely managing threads, your coroutines, or the AWT Event Queue) , trigger this right after liberating your 1MB pre-allocated emergency Out Of Memory (OOM) reserve space:

```kotlin

    // 1. Free your local 1MB OOM safety zone so the app has breathing room to run code

    // 2. Trigger the silent, automatic background pipeline to your server

    // 3. Fall back to your existing logic (writing local backups / logging to disk)

}

```

## How It Operates in Production

1. **The Crash:** A user runs an edge case on Windows, throwing an unhandled error.
2. 
**The Catch:** `CrashReporter.kt` instantly steps in, clears the emergency RAM memory buffer, and dumps the local files.


3. 
**The Autosend:** The background thread processes the HTTP `POST` to your webhook silently and exits.


4. **The Alert:** Your phone or desktop client pops up a crisp, color-coded diagnostic embed displaying the broken Kotlin line number and the precise stack trace. You are no longer blind.
